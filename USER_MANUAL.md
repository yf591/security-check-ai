## はじめに

### セキュリティチェックAIとは

セキュリティチェックAIは、過去のセキュリティチェックシートやQ&A資料を学習し、新しいセキュリティ質問に対して適切な回答を自動的に提案する営業支援ツールです。

### できること

- ✅ 過去のセキュリティチェック資料（PDF、Word、Excel等）からQ&Aデータベースを自動構築
- ✅ 質問を入力すると、類似する過去の質問と回答を瞬時に検索
- ✅ 複数の質問を一括処理して、まとめて回答を取得
- ✅ 検索結果をCSVファイルでダウンロード
- ✅ 検索履歴の管理と共有

### こんな方におすすめ

- セキュリティチェックシートの回答作成に時間がかかっている営業担当者
- 過去の回答資料を探すのに苦労している方
- チーム全体でセキュリティ回答の品質を統一したい方

## インストール方法

### Windows の場合

#### ステップ1: Pythonのインストール

1. https://www.python.org/downloads/ にアクセス
2. 最新版のPythonをダウンロード
3. インストーラーを実行
4. **重要**: 「Add Python to PATH」にチェックを入れる
5. 「Install Now」をクリック

#### ステップ2: プロジェクトの配置

1. `security-check-ai` フォルダを任意の場所に配置
   - 例: `C:\Users\YourName\Documents\security-check-ai`

#### ステップ3: 仮想環境の作成

1. コマンドプロンプトを開く（`Win + R` → `cmd`）
2. プロジェクトフォルダに移動
   ```cmd
   cd C:\Users\YourName\Documents\security-check-ai
   ```
3. 仮想環境を作成
   ```cmd
   python -m venv .venv
   ```

#### ステップ4: パッケージのインストール

1. 仮想環境をアクティベート
   ```cmd
   .venv\Scripts\activate
   ```
2. 必要なパッケージをインストール
   ```cmd
   pip install -r requirements.txt
   ```
   ※ 初回は5-10分程度かかります

### macOS / Linux の場合

#### ステップ1: Pythonの確認

```bash
python3 --version
```
Python 3.8以上であることを確認

#### ステップ2: プロジェクトの配置

`security-check-ai` フォルダを任意の場所に配置

#### ステップ3: 仮想環境の作成とパッケージインストール

```bash
cd /path/to/security-check-ai
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 基本的な使い方

### 初回セットアップ（データの準備）

#### サンプルデータで試す

1. アプリを起動
2. 画面左側のサイドバーで「🔄 data/rawから再構築」ボタンをクリック
3. サンプルデータ（15件のQ&A）が自動的に読み込まれます
4. 数分待つと「データベース構築完了」のメッセージが表示されます

#### 独自のデータを追加する

**方法1: ファイルアップロード**

1. サイドバーの「ファイルをアップロード」セクションを開く
2. 「Browse files」をクリック
3. 以下のいずれかのファイルを選択：
   - PDF (.pdf)
   - Word文書 (.docx)
   - Excelファイル (.xlsx, .xls)
   - CSVファイル (.csv)
   - テキストファイル (.txt)
4. 「📥 ファイルを処理してDBに追加」ボタンをクリック
5. 処理が完了するまで待つ

**方法2: フォルダに直接配置**

1. `data/raw/` フォルダに資料ファイルをコピー
2. サイドバーの「🔄 data/rawから再構築」ボタンをクリック
3. 全ファイルが一括で処理されます

### 質問の検索

#### 単一の質問を検索

1. 「🔍 質問検索」タブを開く
2. テキストエリアに質問を入力
   ```
   例: データの暗号化方法について教えてください
   ```
3. 必要に応じて設定を調整：
   - **検索結果数**: 表示する回答の数（1-10）
   - **類似度閾値**: 検索精度の調整（0.5推奨）
4. 「🔍 検索」ボタンをクリック
5. 結果が表示されます

#### 検索結果の見方

各回答には以下の情報が表示されます：

- **類似度スコア**: 質問との類似度（100%に近いほど関連性が高い）
- **質問**: データベース内のマッチした質問
- **回答**: その質問に対する回答
- **出典**: 元になった資料ファイル名

結果は類似度の高い順に表示されます。

### 一括質問処理

一括質問処理には2つのモードがあります：

#### 処理モードの選択

**質問リスト処理モード**
- 既存の質問に対する回答を一括で検索
- セキュリティチェックシートへの回答作成に最適
- 例：顧客からのセキュリティ質問リストに対して回答を生成

**資料チェックモード（段落抽出）**
- 営業資料や提案書から自動的に段落を抽出
- 各段落が既知のセキュリティ質問に類似していないかチェック
- 機密情報の混入チェックに最適
- 例：提案資料に機密情報が含まれていないか事前チェック

#### 入力方法

**方法1: テキスト入力**

質問リスト処理の場合：
1. 「📝 一括質問処理」タブを開く
2. 処理モードで「質問リスト処理」を選択
3. 「テキスト入力」を選択
4. テキストエリアに質問を入力（1行に1質問）
   ```
   データの暗号化方法について
   アクセス制御の設定方法
   バックアップの頻度について
   ```

資料チェックの場合：
1. 処理モードで「資料チェック（段落抽出）」を選択
2. 「テキスト入力」を選択
3. チェックしたい資料のテキストを貼り付け
4. 自動的に段落ごとに分割されてチェックされます

**方法2: ファイルアップロード**

対応ファイル形式：
- **CSV** (.csv): 表形式データ
- **Excel** (.xlsx, .xls): Excelワークブック
- **PDF** (.pdf): PDFドキュメント
- **Word** (.docx): Word文書
- **テキスト** (.txt): プレーンテキスト

質問リスト処理の場合：
1. 「ファイルアップロード」を選択
2. 質問を含むファイルをアップロード
3. CSV/Excelの場合：質問が含まれる列を選択
4. PDF/Word/テキストの場合：自動的に行または段落を抽出

資料チェックの場合：
1. 「資料チェック（段落抽出）」を選択
2. チェックしたい営業資料をアップロード
3. 自動的に段落を抽出してセキュリティチェック

#### 設定の調整

**質問リスト処理モードの場合：**
- **各質問の検索結果数**: 1-5（通常は1、複数の回答候補が欲しい場合は3-5）
- **類似度閾値**: 0.6推奨（精度重視）

**資料チェックモードの場合：**
- **各段落の検索結果数**: 1-3（通常は1）
- **類似度閾値（機密情報検出感度）**: 0.4推奨（幅広くチェック）
  - 低め（0.3-0.4）：見逃しを防ぐ、幅広く検出
  - 高め（0.6-0.7）：明確に類似する項目のみ検出

#### 結果の見方

**質問リスト処理の結果：**

| 列名 | 説明 |
|------|------|
| 元の質問/テキスト | 入力した質問 |
| マッチした質問 | データベース内の類似質問 |
| 回答 | その質問に対する回答 |
| 類似度 | 質問の類似度（高いほど関連性が高い） |
| 出典 | 回答の元となった資料 |

**資料チェックモードの結果：**

| 列名 | 説明 |
|------|------|
| チェック対象段落 | 資料から抽出された段落 |
| 類似する既知の質問 | 既知のセキュリティ質問で類似するもの |
| セキュリティ観点 | セキュリティ上の観点や注意点 |
| 関連度 | 既知の質問との関連度（高いほど要注意） |
| 参照元 | 参考となる既知のQ&A資料 |

💡 **ヒント（資料チェック）**: 関連度が高い項目は、既知のセキュリティ質問に類似しているため、機密情報が含まれている可能性があります。特に0.5以上の項目は要確認です。

#### 結果のダウンロード

1. 処理完了後、結果が表形式で表示されます
2. 「📥 結果をCSVでダウンロード」ボタンをクリック
3. ファイル名：
   - 質問リスト処理：`security_check_results_YYYYMMDD_HHMMSS.csv`
   - 資料チェック：`document_security_check_YYYYMMDD_HHMMSS.csv`

1. 「📊 履歴」タブを開く

表示内容：
- タイムスタンプ
- 検索クエリ
- マッチした質問

- 出典

1. 「📊 履歴」タブを開く
2. 「履歴をCSVでダウンロード」ボタンをクリック

「🗑️ 履歴をクリア」ボタンをクリックすると、表示中の履歴が削除されます。

### 検索精度の調整

#### 検索結果数

- **設定値**: 1-10
- **推奨**: 単一質問は3-5、一括処理は1-2
- **効果**: 多いほど選択肢が増えるが、確認に時間がかかる

#### 類似度閾値

- **設定値**: 0.0-1.0
- **推奨**: 0.5-0.6
- **効果**: 
  - 低い（0.3-0.5）: 幅広く検索、精度は下がる
  - 中程度（0.5-0.7）: バランスが良い（推奨）
  - 高い（0.7-1.0）: 精度重視、該当なしが増える

## よくある質問

### Q1: どのようなファイルに対応していますか？

**A**: データベース構築と一括処理の両方で以下のファイル形式に対応しています：

**データベース構築用（Q&A資料）：**
- **PDF** (.pdf): セキュリティチェックシートや回答集
- **Microsoft Word** (.docx): 社内セキュリティガイドライン
- **Microsoft Excel** (.xlsx, .xls): Q&A一覧表
- **CSV** (.csv): 質問と回答のリスト
- **テキストファイル** (.txt): Q&A形式のテキスト

**一括質問処理用（入力ファイル）：**
- **CSV/Excel**: 顧客からのセキュリティ質問リスト
- **PDF/Word**: 提案書や営業資料（資料チェック用）
- **テキスト**: 任意のテキストデータ

### Q2: Q&Aのフォーマットはどうすればいいですか？

**A**: 以下のいずれかの形式で記述されていれば自動認識します：

```
Q: 質問文
A: 回答文

質問: 質問文
回答: 回答文

【質問】質問文
【回答】回答文
```

ExcelやCSVの場合、「質問」「Q」「回答」「A」といった列名があれば自動的に認識します。

### Q3: インターネット接続は常に必要ですか？

**A**: 初回起動時のみ必要です（埋め込みモデルのダウンロード）。以降はオフラインで使用できます。

### Q4: データはどこに保存されますか？

**A**: 全てのデータはローカルPC内の以下のフォルダに保存されます：
- 元データ: `data/raw/`
- ベクトルデータベース: `vectordb/`

外部サーバーには一切送信されません。

### Q5: 複数人で同時に使えますか？

**A**: 1つのアプリケーションインスタンスは1人が使用します。複数人で使う場合は：
- 各自のPCにインストール
- または、共有サーバーで起動して異なるポートで複数立ち上げ

### Q6: データを追加したら自動的に学習されますか？

**A**: はい。ファイルをアップロードまたは「再構築」を実行すると、自動的にデータベースに追加されます。

### Q7: 精度を上げるにはどうすればいいですか？

**A**: 以下の方法で精度が向上します：
1. **より多くのQ&Aデータを登録**（100件以上推奨）
   - データ量が多いほど、様々なパターンの質問に対応できます
2. **質問と回答が明確に区別されたデータを使用**
   - Q&A形式が明確なファイルを優先的に登録
3. **類似度閾値を適切に調整**
   - 質問検索：0.5-0.6推奨（バランス重視）
   - 資料チェック：0.4推奨（見逃し防止）
4. **業界や用途に特化したデータを登録**
   - 自社のセキュリティチェック実績を積極的に登録

### Q8: 検索結果が表示されません

**A**: 以下を確認してください：
1. データベースにデータが登録されているか（サイドバーの件数を確認）
2. 類似度閾値が高すぎないか（0.5程度に調整）
3. 質問の言い回しを変えてみる

### Q9: 動作が遅いです

**A**: 以下を試してください：
1. 検索結果数を減らす（3以下）
2. データベースの件数を確認（数千件以上ある場合は遅くなる可能性）
3. PCのメモリ使用状況を確認

### Q10: エラーが出て起動しません

**A**: トラブルシューティングのセクションを参照してください。

### Q11: 資料チェックモードはどんな時に使いますか？

**A**: 以下のような場面で使用します：
- **提案書の事前チェック**: 顧客に提出する前に、機密情報が含まれていないか確認
- **営業資料のレビュー**: 社外に出す資料に社内情報が混入していないか確認
- **セキュリティリスク評価**: 文書内容が既知のセキュリティ質問に該当しないか確認
- **コンプライアンスチェック**: 外部公開資料の適切性を確認

関連度が0.5以上の項目は特に注意が必要です。

### Q12: PDFやWordファイルから質問が正しく抽出されません

**A**: 以下を確認してください：
1. **ファイルの種類**
   - テキストが選択できるPDF/Wordファイルか確認
   - 画像化されたPDFは非対応（OCR処理が必要）
2. **処理モードの選択**
   - 質問リスト：行単位で抽出
   - 資料チェック：段落単位で抽出
3. **ファイルの構造**
   - 明確な段落分けや改行があるか確認
   - 表形式の場合はExcelまたはCSVに変換

### Q13: 一括処理で対応できるファイルサイズの上限は？

**A**: 
- **推奨サイズ**: 1ファイルあたり10MB以下
- **最大処理数**: 一度に1000件程度の質問/段落まで
- 大きなファイルの場合：
  - ファイルを分割して処理
  - PDFは数十ページ程度に分割
  - Excelは数百行ごとに分割

### Q14: 類似度が低くても重要な情報がある場合は？

**A**: 
1. **類似度閾値を下げる**（0.3-0.4程度）
2. **データベースにQ&Aを追加**
   - 重要な質問と回答をデータベースに登録
   - 類似パターンを増やすことで検出精度向上
3. **キーワード検索を併用**
   - 重要なキーワードを含む段落を手動で確認

## トラブルシューティング

### 起動時のエラー

#### エラー: "Python was not found"

**原因**: Pythonがインストールされていない、またはPATHが通っていない

**解決方法**:
1. Pythonを再インストール
2. インストール時に「Add Python to PATH」にチェック
3. コマンドプロンプトを再起動

#### エラー: "No module named 'streamlit'"

**原因**: パッケージがインストールされていない

**解決方法**:
```cmd
# Windowsの場合
.venv\Scripts\activate
pip install -r requirements.txt

# Mac/Linuxの場合
source .venv/bin/activate
pip install -r requirements.txt
```

#### エラー: "Address already in use"

**原因**: ポート8501が既に使用されている

**解決方法**:
別のポートで起動する
```cmd
streamlit run app.py --server.port 8502
```

### 表示の問題

#### 日本語が文字化けする（Windows）

**解決方法**:
コマンドプロンプトで以下を実行してから起動
```cmd
chcp 65001
```

または `start_app.bat` を使用（自動的に設定されます）

#### ブラウザが自動で開かない

**解決方法**:
手動でブラウザを開き、以下のURLにアクセス
```
http://localhost:8501
```

### データ処理の問題

#### ファイルをアップロードできない

**解決方法**:
1. ファイルサイズを確認（大きすぎる場合は分割）
2. ファイル形式が対応しているか確認
3. ファイルが破損していないか確認

#### Q&Aが正しく抽出されない

**解決方法**:
1. ファイル内のQ&Aフォーマットを確認
2. Excelの場合、「質問」「回答」などの列名を使用
3. テキストが読み取り可能か確認（画像化されたPDFは不可）

#### データベース構築が進まない

**解決方法**:
1. 初回起動時は埋め込みモデルのダウンロードで5-10分かかります
2. ブラウザをリロードしてみる
3. ターミナル/コマンドプロンプトにエラーが表示されていないか確認

### パフォーマンスの問題

#### 検索が遅い

**解決方法**:
1. データベースの件数を確認（数千件以上ある場合は分割を検討）
2. 検索結果数を減らす
3. PCのメモリを確認（8GB以上推奨）

#### メモリ不足エラー

**解決方法**:
1. 他のアプリケーションを終了
2. データベースを分割（カテゴリ別など）
3. より大きなメモリを持つPCを使用

### データベースのリセット

全てを初期状態に戻す場合：

**Windows**:
```cmd
rmdir /s vectordb
del data\raw\*.* （サンプル以外）
```

**Mac/Linux**:
```bash
rm -rf vectordb/
rm data/raw/* （.gitkeepは残す）
```

その後、アプリを再起動してデータを再登録してください。

## 付録

### A. 用語集

#### 基本用語

**RAG (Retrieval-Augmented Generation)**
- 検索拡張生成。既存のデータベースから関連情報を検索して回答を生成する技術
- このアプリでは、過去のQ&Aを検索する部分に使用

**ベクトルデータベース**
- テキストを数値ベクトルに変換して保存するデータベース
- 意味が似ているテキストを高速に検索できる
- ChromaDBを使用

**埋め込みモデル (Embedding Model)**
- テキストを数値ベクトルに変換するAIモデル
- all-MiniLM-L6-v2を使用（CPU対応の軽量モデル）

**類似度スコア**
- 0.0〜1.0の値で、質問の類似度を表す
- 1.0に近いほど似ている、0.0に近いほど関係ない

**類似度閾値**
- 検索結果として表示する最低限の類似度
- これより低い類似度の結果は表示されない

#### 処理モード用語

**質問リスト処理**
- 既存の質問に対して回答を検索するモード
- セキュリティチェックシートへの回答作成に使用

**資料チェック（段落抽出）**
- ドキュメントから段落を自動抽出してチェックするモード
- 営業資料の機密情報チェックに使用

**段落抽出**
- 文章を意味のあるまとまり（段落）に自動分割すること
- 空行や改行パターンで判断

**機密情報チェック**
- 資料に含まれる内容が、既知のセキュリティ質問に類似していないか確認
- 関連度が高い場合、機密情報が含まれる可能性がある

#### データベース用語

**Q&Aペア**
- 質問と回答の組み合わせ
- データベースの基本単位

**出典 (Source)**
- Q&Aペアの元となったファイル名
- 回答の信頼性確認に使用

**データベース再構築**
- 既存のデータベースを削除して、新しく作り直すこと
- data/rawフォルダ内の全ファイルを再処理

#### ファイル形式用語

**CSV (Comma-Separated Values)**
- カンマ区切りのテキストファイル
- Excelで開ける表形式データ

**PDF (Portable Document Format)**
- 文書のレイアウトを保持するファイル形式
- テキスト抽出に対応（画像化されたPDFは非対応）

**DOCX**
- Microsoft Word 2007以降の文書形式
- 段落やテーブルの構造を保持

**XLSX**
- Microsoft Excel 2007以降のワークブック形式
- 複数シートに対応
